# 레시피 뷰어

DummyJSON API를 활용하여 레시피를 조회하고 검색할 수 있는 웹 애플리케이션입니다. React와 Vite를 사용하여 구현했습니다.

## 기술 스택

- React 19.1.1: 컴포넌트 기반 UI 라이브러리
- Vite: 빠른 개발 서버 및 빌드 도구
- DummyJSON API: 레시피 데이터 제공

## 주요 기능

레시피 목록 조회
- DummyJSON API를 통한 레시피 데이터 가져오기
- 페이지당 12개 레시피 표시
- 레시피 이미지, 이름, 난이도, 소요 시간, 인분, 평점 표시
- 태그 정보 표시

레시피 검색
- 레시피 이름으로 검색
- 검색 결과 페이지네이션
- 검색 초기화 기능

페이지네이션
- 페이지 번호 네비게이션
- 최대 5개 페이지 번호 표시
- 생략 표시(...)를 통한 긴 페이지 목록 처리
- 이전/다음 버튼

레시피 상세 모달
- 레시피 카드 클릭 시 상세 정보 모달 표시
- 재료 목록 표시
- 조리 방법 단계별 표시
- 칼로리, 요리 종류, 난이도, 리뷰 수 등 추가 정보
- 식사 유형 태그 표시

## 구현 세부사항

레시피 데이터는 DummyJSON API의 /recipes 엔드포인트를 사용하여 가져옵니다. fetch API를 사용하여 비동기로 데이터를 로드하며, limit과 skip 파라미터를 통해 페이지네이션을 구현했습니다.

검색 기능은 /recipes/search 엔드포인트를 사용하며, 검색어를 encodeURIComponent로 인코딩하여 URL에 포함시킵니다. 검색 모드와 일반 모드를 isSearching 상태로 구분하여 관리합니다.

페이지네이션 로직은 getPageNumbers 함수에서 처리합니다. 현재 페이지 위치에 따라 표시할 페이지 번호를 계산하며, 총 페이지 수가 5개 이하일 때는 모두 표시하고, 그 이상일 때는 현재 페이지 주변과 첫 페이지, 마지막 페이지만 표시합니다.

모달은 selectedRecipe 상태를 통해 관리합니다. 레시피 카드 클릭 시 해당 레시피 정보를 상태에 저장하고, 모달 오버레이 클릭 시 상태를 null로 설정하여 닫습니다. stopPropagation을 사용하여 모달 내용 클릭 시 닫히지 않도록 처리했습니다.

로딩 상태는 loading 상태로 관리하며, 데이터를 가져오는 동안 스피너를 표시합니다. 에러 발생 시 error 상태에 메시지를 저장하고, 에러 메시지와 재시도 버튼을 표시합니다.

페이지 변경 시 window.scrollTo를 사용하여 페이지 상단으로 부드럽게 스크롤합니다.
